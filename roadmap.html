P2
APK File Tree + Manifest XML Viewer
Decoded APK structure display with readable manifest
📋 PS Requirement🔴 CriticalDay 1–2
▼
🌳
backend: extract_file_tree()
Parse APK ZIP structure → nested JSON tree. Flag suspicious: *.so in assets, hidden DEX files.
📋
backend: get_manifest_xml()
Return decoded, human-readable AndroidManifest.xml string from existing manifest_parser.py.
🖥️
APKFileTree.tsx (Frontend)
Recursive collapsible tree UI. Click any file → open viewer. ⚠️ flags on suspicious entries.
📄
ManifestViewer.tsx (Frontend)
Syntax-highlighted XML. Highlights <uses-permission>, <service>, <receiver> in different colors.
New API endpoints: GET /api/analysis/{id}/filetree and GET /api/analysis/{id}/manifest
P3
PCAP Upload + Network Traffic Analysis
Parse network captures for C2 detection
📋 PS Requirement🔴 CriticalDay 2–3
▼
📡
POST /api/upload/pcap
Accept .pcap / .pcapng files. Validate extension, save to S3, enqueue analysis task.
🔬
backend/engines/pcap_analyzer.py
Extract: unique IPs, domains, HTTP hosts, TLS SNI, suspicious ports, data volumes per IP.
🔑
JA3 TLS Fingerprinting
Compute JA3 hash from TLS ClientHello bytes. Match against known malware fingerprint list.
📂
PCAP Drop Zone (Frontend)
Second upload area in DropZone.tsx — "Optional: Drop PCAP for extended analysis".
Dependencies: dpkt (pip install dpkt) for PCAP parsing. JA3 blocklist JSON from known malware TLS fingerprints.
P4
Celery + Redis + WebSocket Progress
Async job queue — biggest architectural upgrade
🔴 CriticalInfraDay 3–4
▼
⚙️
backend/worker/celery_app.py
Celery factory with Upstash Redis as broker. task_track_started = True.
📋
backend/worker/tasks.py
run_full_analysis() task — stages: manifest→yara→strings→intel→ai→scoring. Emits Redis progress.
🔌
backend/routes/websocket.py
WS /ws/progress/{scan_id} — reads Redis state, pushes {stage, pct} to client every 500ms.
📤
Refactor upload.py
Change from inline analysis → run_full_analysis.delay(). Return scan_id immediately.
🏗️ Everything downstream (MobSF, C2 detection, ML classifier) depends on async task infrastructure. Do this before P5–P7.
P5
Database + Storage Upgrade
SQLite → production data layer (all free tier)
InfraDay 4–5
▼
🐘
PostgreSQL — Supabase
Scan metadata, IOC DB, structured results. Alembic migrations.
🍃
MongoDB Atlas
backend/db/mongo.py — store full raw JSON analysis blobs per scan.
☁️
AWS S3 / Cloudflare R2
backend/storage/s3.py — upload APKs, PCAPs, PDFs. Return signed URLs (15min TTL).
🔍
Bonsai Elasticsearch
backend/db/elastic.py — index IOCs per scan. GET /api/search?q= endpoint.
All four services are free tier — combined cost: $0/month. PostgreSQL keeps SQLite as dev fallback via env flag.
P6
JADX Decompilation + Code Viewer
Java source from DEX — mentor reference tool
📋 PS RequirementDay 5–6
▼
☕
backend/engines/jadx_decompiler.py
Subprocess call to jadx CLI. Output: per-class Java source + suspicious API flag list.
🚩
find_suspicious_classes()
Flag classes containing: Cipher, SMS, exec(), Runtime, OkHttp, TelephonyManager APIs.
🖥️
CodeViewer.tsx (Frontend)
VS Code-style UI — file tree left, syntax-highlighted Java right. Red highlight on suspicious APIs.
JADX binary available at: https://github.com/skylot/jadx/releases — add to Docker image via curl download in Dockerfile.
P7
MobSF Dynamic Sandbox
Runtime APK detonation with Frida hooks
📋 PS Requirement🔴 CriticalDay 5–7
▼
🐳
backend/engines/dynamic_analyzer.py
MobSF REST API client — upload, start scan, poll status, fetch JSON report, download PCAP.
🪝
backend/engines/frida_hooks.py
Parse Frida hook logs: crypto calls, SMS ops, file I/O, network requests, contact access.
📊
DynamicResultsCard.tsx
Display dynamic findings: API calls made, files created, network requests, SMS sent.
⚠️ MobSF is heavy. For demo: use mobsf.live API (free hosted). For production: Docker on EC2 t3.medium+. Skip Android emulator on free tier.
P8
Advanced C2 + Threat Intelligence
Beacon, DGA, JA3, OTX, ASN enrichment
📋 PS RequirementDay 6–7
▼
📡
beacon_detector.py
Timing variance analysis on network timestamps. <10% variance = C2 beacon. Returns interval + confidence.
🎲
dga_detector.py
Shannon entropy + label length scoring. Score >0.65 = DGA suspected. Per domain analysis.
🌐
otx.py — AlienVault OTX
IP, domain, file hash lookups against OTX threat intel. Free API key at otx.alienvault.com.
☁️
asn_lookup.py — Hosting Provider
IP → ASN, org, country, city. Flag datacenter IPs (AWS, GCP, Alibaba, Vultr). Free ipapi.co.
🔬
VirusTotal Full Upgrade
Add URL scan + IP lookup to existing virustotal.py. Currently hash-only.
🇮🇳
India IOC DB — 200+ Entries
Migrate to PostgreSQL table. Add admin API (POST/DELETE /api/admin/ioc) with API key auth.
P9
Static ↔ Dynamic Correlation Engine
Find hidden C2s — runtime vs hardcoded discrepancies
📋 PS Requirement🔴 CriticalDay 7–8
▼
🔗
backend/engines/correlation_engine.py
Compare static IOC sets vs dynamic network observations. Output: matched, static-only, runtime-only.
🚩
Anomaly Flag Generation
Auto-flag: SMS without SEND_SMS permission, contacts accessed at runtime but not in manifest, DGA domains.
🖥️
CorrelationPanel.tsx (Frontend)
Split view: Static | Dynamic. Green = confirmed, Yellow = static only, Red = runtime anomaly.
This is the feature that most impresses investigators — it directly shows what the app LIED about in its manifest vs what it actually did at runtime.

Court-grade report with chain of custody, methodology, verdict
📋 PS Requirement🔴 CriticalDay 9
▼

Cover Page

🔑 Critical path = what judges will evaluate first
The file tree (P2) is the first thing a judge clicks after upload. The forensic PDF (P13) is what they download at the end. The correlation engine (P9) is the core technical differentiator. Get these three plus the async backbone (P4) right and you win.
